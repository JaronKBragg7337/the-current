# Asset strategy and source inventory

Last researched: 2026-07-13

## Current incorporated-asset status

**No external 3D model, animation, texture, HDRI, audio file, or downloaded source archive is currently incorporated.** [`assets/manifest.json`](../assets/manifest.json) contains zero external asset records and zero runtime files. Every record in [`assets/source-catalog.json`](../assets/source-catalog.json) is explicitly marked `candidate-not-downloaded`, `incorporated: false`, and has no archive hash or runtime file.

Any geometry currently displayed by the application is generated by The Current's own source code. That procedural terrain, road, building, person, vehicle, and prop geometry is project-authored; it must not be represented as content from any candidate pack. The catalog documents possible future inputs, not assets already present in the world.

This distinction is enforced by `npm run assets:validate`. A downloaded source may not remain in the research catalog while a derived file is committed or loaded. It must receive a complete incorporated manifest record, exact hashes, a local license-evidence file, documented transformations, and verified runtime paths first.

## Governing asset decisions

1. **Simulation state is authoritative.** A crop stage, carried box, occupied shop, damaged road, construction frame, or moving vehicle may appear only because corresponding simulation state exists. Cosmetic assets cannot invent activity.
2. **Procedural topology stays project-owned.** Terrain, settlement footprints, road centerlines and surfaces, navigable door placement, construction stages, collision proxies, and placement rules are generated or assembled by The Current. External packs may supply curated visual modules, never the causal or navigation topology.
3. **One coherent character ecosystem comes first.** If the preferred candidates pass a benchmark, Quaternius Universal Base Characters and the two Universal Animation Libraries become the canonical selected/near humanoid rig. Other humanoid animations must be retargeted and baked once to that skeleton.
4. **A pack is not a dependency until its exact files are pinned.** A permissive landing page is research evidence, not proof of the bytes eventually downloaded. The archive's included notice, SHA-256, size, download date, selected files, transformations, and derived hashes must be recorded.
5. **Web delivery controls selection.** Packs are source libraries, not bundles to ship wholesale. Runtime artifacts are curated, atlased, compressed, split into lazy-loaded groups, and assigned measured LODs.
6. **Accessible geometry is real geometry.** Painted or shader-faked windows do not count as interiors. A visible door must have a valid pivot, clearance, collision opening, and navigable connection if NPCs use it.
7. **License simplicity is a feature.** CC0 is preferred. Attribution licenses can be accepted only with an explicit display plan and review. Marketplace licenses that block separable redistribution are not suitable for an inspectable public repository.

### Proposed humanoid rendering tiers

The triangle counts are initial budget hypotheses, not asset claims:

| Tier | Proposed geometry | Animation/update model |
| --- | ---: | --- |
| Selected/near | About 13k triangles from the candidate base, subject to measurement | Full skinned character, accurate local pose, equipment, first-person eye anchor |
| Medium | About 4k–6k triangles | Reduced skin/clip set and lower update frequency |
| Far | About 800–1,500 triangles | Instanced or baked animation representation |
| Orbital | Below 200 triangles, impostor, or point representation | Aggregate motion; no facial or per-limb processing |
| Outside rendered regions | No mesh | Full authoritative simulation continues |

Age is not a sudden model swap. Child/teen/adult/older-adult/elder presentation should use continuous scale and proportion curves, posture, movement speed, hair tint/loss, clothing, and health cues derived offline and parameterized at runtime. The hidden natural lifespan remains simulation-only.

## Sources deliberately excluded from the initial pipeline

- **Mixamo:** Adobe permits use in games, but its licensing guidance prohibits redistribution of raw character and animation files. A public source repository containing separable GLBs creates avoidable risk.
- **BlenderKit and similar royalty-free marketplaces:** separable redistribution restrictions conflict with a repository where optimized runtime assets remain directly inspectable.
- **Sketchfab, OpenGameArt, and Poly Pizza as bulk sources:** licenses vary per item and provenance/takedown burden is high. An exceptional item could be considered only after item-level review; Poly Pizza also prohibits scraping.
- **Ready Player Me and avatar SaaS platforms:** proprietary terms and platform coupling conflict with deterministic offline operation and reusable public assets.
- **Unity Asset Store, Unreal Marketplace, and similar paid packs:** raw/open redistribution is commonly prohibited.
- **MB-Lab:** MakeHuman/MPFB currently has clearer official documentation for exported-model licensing.

Exclusion is an asset-pipeline decision, not a claim that the source is unlawful or low quality.

## Incorporation workflow

1. Open an issue or design note identifying a simulation-visible gap and acceptance budget.
2. Revisit the official landing and license pages. Confirm that the exact archive is downloadable without credentials that would be stored or redistributed.
3. Download manually into ignored `assets/source-cache/`. Never commit browser sessions, cookies, signed URLs, or account data.
4. Record archive filename, exact byte size, SHA-256, actual download date, creator, landing URL, acquisition URL, included license notice, and any restrictions.
5. Select only required source files. Record every selected internal archive path.
6. Normalize and derive artifacts according to [`assets/PIPELINE.md`](../assets/PIPELINE.md).
7. Save an asset-specific license/provenance note under `assets/licenses/` and optimized deliverables under `assets/runtime/`.
8. Move the source out of `source-catalog.json` and add a complete entry to `manifest.json`. A grouped research entry becomes separate incorporated entries when archives or item licenses differ.
9. Run `npm run assets:validate`, glTF validation, visual scale/contact checks, and the relevant browser/performance scenario.
10. Commit the manifest, evidence, scripts or deterministic instructions, and runtime derivatives together. Do not commit the raw archive unless redistribution is necessary, useful, licensed, and intentionally reviewed.

## Initial runtime budgets

These are acceptance targets for the first curated external-asset slice, not current measurements:

- Aim for a **25–40 MB initial nearby-world transfer**, with regional and selected-character content lazy-loaded.
- Prefer one or a few shared atlases per family and measurable draw-call reductions over many bespoke materials.
- A selected/near character should not load all 250+ candidate animation clips.
- Texture sources above the demonstrated on-screen need are downsampled; color textures may use WebP, while tested KTX2/Basis variants can serve GPU-efficient material maps.
- Every model family receives close, medium, far, and where useful orbital representations. LOD generation is reviewed visually, not accepted solely because a decimator completed.
- Collision uses simplified project-authored proxies. Rendering triangles are not navigation collision by default.

## Known asset limitations

- The current external manifest is empty, so the procedural presentation remains visually limited compared with the intended stylized-but-believable standard.
- The proposed Quaternius character stack still needs acquisition, included-license verification, rig inspection, close-camera visual review, and performance measurement.
- Child and elder bodies, contemporary occupation clothing, carried-object contact, facial variation, and facial animation need project-authored derivation or a later generator benchmark.
- No external building kit has yet passed functional-door, interior, collision, construction-stage, and coherent-scale tests.
- No nature, material, vehicle, prop, or animation source has yet been benchmarked in the application.
- No large binaries have been added to Git LFS because no external binary is currently incorporated.

The empty manifest is intentional honesty, not an assertion that asset work is complete.
